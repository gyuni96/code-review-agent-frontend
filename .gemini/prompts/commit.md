# Commit Message Template

다음 형식에 따라 정석적인 Conventional Commit 메시지를 작성하세요.

## Structure

```
<type>[optional scope]: <description>

[optional body]

[optional footer(s)]
```

## Type Guidelines

- `feat`: 새로운 기능 추가
- `fix`: 버그 수정
- `docs`: 문서 변경
- `style`: 코드 포맷팅, 세미콜론 누락 등 (코드 변경 없음)
- `refactor`: 코드 리팩토링 (기능 변화 없음)
- `test`: 테스트 코드 추가/수정
- `chore`: 빌드 업무 수정, 패키지 매니저 설정 등

## Writing Guidelines

1. 요약문은 50자 이내로 간결하게.
2. 요약문 끝에 마침표(.) 사용 금지.
3. 명령형으로 시작할 것 (예: "Add" vs "Added").
4. 본문(Body)에는 "무엇을", "왜" 했는지 구체적으로 설명.
